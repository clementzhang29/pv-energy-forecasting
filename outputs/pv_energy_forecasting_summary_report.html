项目概述

本项目目标是根据设备经纬度、太阳几何、天气特征和时间特征，预测设备每15分钟粒度的发电量 pvGenTotal。最终评估以 单日预测总量 vs 单日真实总量的 MAPE 为核心指标，同时保留15分钟粒度预测以便检查日内曲线形态。

本报告首次系统对比了 深度学习模型 (DNN/CNN/LSTM)、传统机器学习模型 (XGBoost/CatBoost/RF/SVR) 和 15种自定义 Pipeline 优化策略。

传统机器学习模型 (ML)

核心发现：ML 模型中 XGBoost 最佳 (0.2215)，但仍显著落后于经过日级校准的 Pipeline 方案 (E07: 0.1181)。ML 模型仅在点级（15分钟粒度）做回归预测，缺乏日级总量校准这一关键环节。

🏅 XGBoost

Combined MAPE: 0.2215

超参数：n_estimators=500, lr=0.04, max_depth=8, subsample=0.85, colsample_bytree=0.85

评价：ML 中最优，已见设备 unseen 部分表现最稳定 (0.2730)。训练速度中等 (45秒)，适合轻量级基线。作为点级回归方法，结合后期日级校准可接近 pipeline 效果。

参考: scripts/run_full_comparison.py → XGBRegressor

CatBoost

Combined MAPE: 0.2276

超参数：iterations=500, lr=0.04, depth=7

评价：与 XGBoost 接近，对类别特征有原生支持但在本项目中特征已多为数值型。训练速度43秒，模型文件略大。

参考: scripts/run_full_comparison.py → CatBoostRegressor

Random Forest

Combined MAPE: 0.2346

超参数：n_estimators=200, max_depth=18, min_samples_leaf=5

评价：训练最慢 (737秒)，泛化能力与 boosting 差距明显。Unseen/Seen 倍率 (1.55x) 与 XGBoost 相近，但绝对值高出近 6%。

参考: scripts/run_full_comparison.py → RandomForestRegressor

SVR (RBF)

Combined MAPE: 0.8179

超参数：C=10, gamma=scale, epsilon=0.01（子集30000行训练）

评价：SVR 不适合 252 维 × 18 万行的大规模回归。子集采样训练导致效果远差于其他方法。未使用 NaN 兼容的 imputer 前会报错。

参考: scripts/run_full_comparison.py → SVR(kernel='rbf')

深度学习模型 (DL) — 2026-07-02 新增

核心发现：DNN/CNN/LSTM 在本任务中表现最差（Combined MAPE 0.63-0.64），远落后于 boosting 树模型 (0.22) 和自定义 pipeline (0.12)。CPU-only 训练环境下，深度学习方案不作为推荐方向。

DNN (5层 MLP)

Combined MAPE: 0.6324

架构: 256→128→64→32→1, BatchNorm, Dropout(0.2)

训练: 50 epochs, HuberLoss (α=1), Adam(lr=0.001), BatchSize=4096

时间: 217秒 | 早停: 未触发

CNN (1D Conv)

Combined MAPE: 0.6337

架构: Conv1d(1→32→64,k=3) → AdaptAvgPool → FC(64→32→1)

训练: PCA降维至64后训练（原252维内存不足）

时间: 531秒 | 早停: Epoch 33

LSTM (2层)

Combined MAPE: 0.6362

架构: LSTM(64, 2层, dropout=0.2) → FC(64→32→1)

训练: PCA降维至64后训练，未利用时序结构优势

时间: 239秒 | 早停: 未触发

DL 表现差的根因分析：
  1. 特征维度过高：252维特征对MLP过于稀疏，CNN因内存限制被迫PCA降维（信息损失从0.22→0.63）。
  2. 缺少日级校准：DL仅做点级回归，没有类似pipeline的日级总量校准步骤。
  3. 时序结构利用不足：LSTM本可利用时序模式，但PCA降维破坏了原始时序关联结构。
  4. 训练数据有限：18万行对DL不够大（boosting仅需几千行即可收敛）。
  5. CPU-only限制：无GPU支持，模型容量和训练轮次受限。

2026-07-02 新增实验详细分析

E12: Huber 损失 LGBM

Combined MAPE = 0.2249（完全持平 E01）

Huber 损失对离群值理论上更鲁棒，但在此数据上未带来任何提升。LightGBM 默认的回归 objective 已对异常值有较好处理能力。不建议继续在损失函数上投入优化。

E13: Fair 损失 LGBM

Combined MAPE = 0.2236（略好 0.6%）

Fair 损失略微改善了 MAPE 友好性，fair_c=1.0 下对低值有小幅修正。但 0.6% 提升近乎噪声水平，不值得增加模型复杂度。

E14: 加权日级校准

Combined MAPE = 0.1240（比 E05 差 4%）

对低输出日 payload weight=3.0 加大权重，反而降低了整体效果。说明低输出日的高 MAPE 主要是 分母效应（真实值趋近0），不是校准模型需要优先拟合的方向。

E15: 自适应 Guard

Combined MAPE = 0.2243（远差于 E09-E11 的 0.19）

基于训练集分布动态确定阈值（15%分位），在实际 unseen 设备上不适应。冷启动场景需要更保守的固定阈值策略，动态调整反而导致未见设备过收缩。

关键发现与结论

1. Pipeline 远优于纯 ML/DL

最优 pipeline (E07: 0.1181) 比最优 ML 模型 (XGBoost: 0.2215) 提升 46.7%，比最优 DL 模型 (DNN: 0.6324) 提升 81.3%。核心优势：

  • 物理归一化：y_norm = pvGenTotal / (solar_proxy + eps) 消除太阳几何和日时段影响
  • 日级校准：专为日总量 MAPE 设计的二级模型（LightGBM 在日级数据上拟合 log1p 变换）
  • 混合策略：验证集最优混合权重，叠加冷启动 guard 保护

2. ML 方法排名：XGBoost > CatBoost > RF >> SVR

Boosting 方法在表格数据上优势明显。XGBoost (0.2215) 在已见 unseen 上表现最稳定 (0.2730)。但所有 ML 模型均缺乏日级校准的精度增益——ML + 日级校准是潜在的提升方向。

3. 深度学习不适用于当前设置

DNN/CNN/LSTM 表现最差 (0.63+)，除非有 GPU 加速、更多训练数据、合理降维和时序特征工程，否则不作为推荐方向。CNN 因内存限制被迫 PCA 降维后表现与 DNN 几乎一致（0.6337 vs 0.6324），说明 1D Conv 未利用到任何空间先验。

4. 冷启动仍是最大挑战 — Test2 MAPE 高达 0.619

所有实验的 Unseen MAPE 显著高于 Seen。E07 的 Unseen (0.1574) 是 Seen (0.0787) 的 2 倍。Test2（6个全新设备）的 MAPE 高达 0.619，说明冷启动容量估计仍不足。

5. 损失函数优化不划算

Huber/Fair 替代 LGBM 默认损失几乎没有提升 (0.2249 vs 0.2236)，加权日级校准反而劣化 (E14: 0.1240 vs E05: 0.1192)。优化资源应投在冷启动策略和特征工程上。

推荐生产方案：E07 - 验证集混合模型 (Validated Blend)
Combined MAPE = 0.1181 | Seen MAPE = 0.0787 | Unseen MAPE = 0.1574
后续优化方向：冷启动容量代理特征、多折 unseen 验证、分组校准、动态不确定性估计。
